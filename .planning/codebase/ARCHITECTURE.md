# HPCC4J Architecture Reference

## 1. Overall Architectural Pattern

HPCC4J is a **multi-module Maven library** (`groupId: org.hpccsystems`) providing Java APIs for interacting with HPCC Systems clusters. Version `10.0.33-0-SNAPSHOT` at time of writing; targets Java 8 (`maven.compiler.release=8`).

The project follows a strict **layered architecture** where lower layers never depend on higher layers, and higher layers depend only on lower layers. The four active modules (declared in the root `pom.xml`) form a directed acyclic dependency graph:

```
commons-hpcc  вҶҗв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
     вҶ‘                                           в”Ӯ
   wsclient  вҶҗв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ  в”Ӯ
     вҶ‘                                        в”Ӯ  в”Ӯ
  dfsclient  вҶҗв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”ҳ
     вҶ‘                                        
  spark-hpcc  (depends on all three above)
```

`clienttools` and `rdf2hpcc` exist in the repository but are **not declared as modules** in the root `pom.xml` вҖ” they are effectively standalone sub-projects.

## 3. Layer Breakdown (wsclient)

The wsclient module uses a strict three-layer architecture to isolate callers from WSDL interface changes.

### Layer 1 вҖ” Generated Axis2 ADB Stubs
**Package:** `org.hpccsystems.ws.client.gen.axis2.<service>.<version>`  
**Location:** `wsclient/src/main/java/org/hpccsystems/ws/client/gen/axis2/`

Raw WSDL-generated classes produced by the Axis2 code generator (tooled via `wsclient/utils/wsInterfaceUpdater.py`). **Never modify these directly.** Multiple historical versions of each service are stored on disk, but most are **excluded from compilation** via `<excludes>` in `wsclient/pom.xml`. Only the `latest` sub-package of each service is compiled.

Services covered:
`filespray`, `wsattributes`, `wscloud`, `wscodesign`, `wsdfu`, `wsdfuxref`, `wsfileio`, `wspackageprocess`, `wsresources`, `wssmc`, `wssql`, `wsstore`, `wstopology`, `wsworkunits`

Example: `org.hpccsystems.ws.client.gen.axis2.wsdfu.latest.WsDfuStub`

### Layer 2 вҖ” Programmatically-Generated Wrapper Classes
**Package:** `org.hpccsystems.ws.client.wrappers.gen.<service>`  
**Location:** `wsclient/src/main/java/org/hpccsystems/ws/client/wrappers/gen/`

These wrappers are generated by `Axis2ADBStubWrapperMaker` (a tool class in `org.hpccsystems.ws.client.utils`). They provide a stable Java API that mirrors the stub types but is insulated from upstream interface changes. All generated wrapper class names follow the pattern `<OriginalClass>Wrapper`.

### Layer 2b вҖ” Hand-Written Wrapper Classes
**Package:** `org.hpccsystems.ws.client.wrappers` (root)  
`org.hpccsystems.ws.client.wrappers.wsdfu`  
`org.hpccsystems.ws.client.wrappers.wsworkunits`

Hand-crafted for types that required special handling (e.g., `DFUFileDetailWrapper`, `DFUFileAccessInfoWrapper`, `EspSoapFaultWrapper`, `ArrayOfEspExceptionWrapper`, etc.). These live in `wsclient/src/main/java/org/hpccsystems/ws/client/wrappers/`.

### Layer 3 вҖ” High-Level Service Client Classes
**Package:** `org.hpccsystems.ws.client`  
**Location:** `wsclient/src/main/java/org/hpccsystems/ws/client/`

Hand-written, service-specific clients that orchestrate calls to the stubs via the wrapper layer:

| Class | Service |
|---|---|
| `HPCCWsClient` | Convenience aggregate facade |
| `BaseHPCCWsClient` | Abstract base for all service clients |
| `HPCCWsDFUClient` | Distributed File Utilities |
| `HPCCWsWorkUnitsClient` | ECL Workunits |
| `HPCCWsSMCClient` | System Management Console |
| `HPCCWsTopologyClient` | Cluster topology |
| `HPCCFileSprayClient` | File spray/despray |
| `HPCCWsResourcesClient` | Roxie resources |
| `HPCCWsStoreClient` | ESP Store |
| `HPCCWsPackageProcessClient` | Package process |
| `HPCCWsCodeSignClient` | Code signing |
| `HPCCWsCloudClient` | Cloud/containerized HPCC |
| `HPCCWsDFUXRefClient` | DFU cross-reference |
| `HPCCWsSQLClient` | HPCC SQL |
| `HPCCWsFileIOClient` | File I/O |
| `extended/HPCCWsAttributesClient` | WsAttributes (crafted, not stub-wrapped) |

## 4. Key Abstractions

### `Connection`
`org.hpccsystems.ws.client.utils.Connection`

Central value object encapsulating all connectivity state:
- Target URL (protocol, host, port)
- Credentials (username/password, stored as a nested `Credentials` inner class)
- HTTP/HTTPS selection and TLS configuration
- Preemptive auth via `HPCCPreemptiveAuthInterceptor`
- OpenTelemetry span injection for outbound HTTP calls
- Raw `sendGetRequest(String path)` for non-SOAP calls (e.g., `esp/getauthtype`)

```java
Connection conn = new Connection("http", "myhpcc.example.com", "8010");
conn.setCredentials("username", "password");
```

### `DataSingleton`
`org.hpccsystems.ws.client.utils.DataSingleton`  extends `java.util.Observable`

Abstract base for objects that maintain state through a background monitoring thread. Used by `BaseHPCCWsClient` (and therefore all service clients) to observe and refresh data lazily. Provides a `monitor()` method that starts a daemon thread polling `isComplete()`.

### `DataSingletonCollection`
`org.hpccsystems.ws.client.utils.DataSingletonCollection`

An in-memory cache / flyweight registry of `DataSingleton` instances. `HPCCWsClient.All` and `HPCCWsClient.SubClients` are static instances of this class, enabling canonical per-connection instances:

```java
// Returns cached instance if one with identical connection already exists
HPCCWsClient client = HPCCWsClient.get(conn);
```

### `BaseHPCCWsClient`
`org.hpccsystems.ws.client.BaseHPCCWsClient` extends `DataSingleton`

Abstract parent of all individual service clients. Responsibilities:
- Holds a `Connection wsconn` reference
- Creates and configures Axis2 `Stub` instances (endpoint, credentials headers, HTTP transport)
- Lazily initialises target platform version (`Version targetHPCCBuildVersion`)
- Detects container mode and auth mode from live ESP queries
- `@WithSpan` annotated methods for OpenTelemetry trace propagation

### `HPCCWsClient`
`org.hpccsystems.ws.client.HPCCWsClient` extends `DataSingleton`

Convenience aggregate that composes sub-clients for all services and exposes the most common operations through a single entry point. Maintains its own `DataSingletonCollection All` and `SubClients` static caches. Does **not** extend `BaseHPCCWsClient` вҖ” it composes service clients rather than inheriting.

### `ObjectPool<T>`
`org.hpccsystems.ws.client.utils.ObjectPool`

Generic timed object pool with `checkOut()` / `checkIn()` semantics. Uses two `Hashtable<T, Long>` maps: `locked` and `unlocked`. Objects in the `unlocked` map expire after `expirationTime` milliseconds and are discarded on next access.

### `HPCCWsClientPool`
`org.hpccsystems.ws.client.HPCCWsClientPool` extends `ObjectPool<HPCCWsClient>`

Concrete pool implementation for `HPCCWsClient` instances. Overrides `create()` to produce `new HPCCWsClient(m_connection)`. Default expiry: 60 seconds. `validate()` and `expire()` hooks available for health checking before re-use.

```java
HPCCWsClientPool pool = new HPCCWsClientPool(conn);
HPCCWsClient client = pool.checkOut();
try { client.doSomething(); }
finally { pool.checkIn(client); }
```

### `FieldDef`
`org.hpccsystems.commons.ecl.FieldDef` implements `Serializable`

The core schema description type. Represents a named field with an HPCC `FieldType` and `HpccSrcType`, optional child `FieldDef[]` for nested RECORD/DATASET/SET fields, size/sign/blob metadata. Serialisable so it can cross JVM boundaries in distributed Spark jobs.

### `RecordDefinitionTranslator`
`org.hpccsystems.commons.ecl.RecordDefinitionTranslator`

Parses the JSON schema payload returned by the HPCC ESP (WsDFU) into a `FieldDef` tree. This JSON schema is the bridge between HPCC's binary wire format and the Java type system. Handles all HPCC ECL type IDs, flag masks, and composite types.

## 5. Data Flow: Typical Web Service Call

```
Caller
  в”Ӯ
  в–ј
HPCCWsClient.get(conn)           вҶҗ retrieves or creates from DataSingletonCollection.All
  в”Ӯ
  в–ј

  в”Ӯ
  в”ңв”Җ Constructs DFUInfoRequest (gen.axis2.wsdfu.latest or wrapper)
  в”Ӯ
  в”ңв”Җ BaseHPCCWsClient.getWsDFUStub()
  в”Ӯ     в””в”Җ Creates WsDfuStub(endpointUrl)
  в”Ӯ     в””в”Җ Sets Axis2 Options (credentials, timeouts, HTTP headers)
  в”Ӯ     в””в”Җ Injects OpenTelemetry W3C trace context headers
  в”Ӯ
  в”ңв”Җ stub.DFUInfo(request)   вҶҗ Axis2 serialises to SOAP/XML, sends HTTP POST
  в”Ӯ                              вҶ’ ESP server at http://host:8010/WsDfu
  в”Ӯ                              вҶҗ SOAP XML response
  в”Ӯ                          вҶҗ Axis2 deserialises to DFUInfoResponse (gen class)
  в”Ӯ
  в”ңв”Җ Wraps DFUInfoResponse вҶ’ DFUFileDetailWrapper (hand-crafted wrapper)

```

### Reading HPCC вҶ’ Spark DataFrame

```

         в–ј

         в”Ӯ
         в–ј

         в”Ӯ
         в–ј
HpccFile(fileName, Connection)   вҶҗ extends dfsclient.HPCCFile
         в”Ӯ
         в”ңв”Җ HPCCWsDFUClient.getFileAccessInfo(fileName, ...)
         в”Ӯ      в””в”Җ ESP returns DFUFileAccessInfoWrapper
         в”Ӯ              (includes file part addresses + access tokens)
         в”Ӯ
         в”ңв”Җ RecordDefinitionTranslator.parseRecord(jsonSchema)
         в”Ӯ      вҶ’ FieldDef tree (canonical HPCC schema)
         в”Ӯ
         в””в”Җ DataPartition[] (one per Thor physical file part)
                    в”Ӯ
                    в–ј
         HpccRDD(SparkContext, DataPartition[], FieldDef)
                    в”Ӯ extends RDD<Row>
                    в”Ӯ maps DataPartitions вҶ’ InternalPartition[] (Spark Partition)
                    в”Ӯ
                    в–ј (per Spark executor / partition)
         HpccRemoteFileReader<Row>(DataPartition, GenericRowRecordBuilder)
                    в”Ӯ
                    в”ңв”Җ RowServiceInputStream (TCP connection to Thor node)
                    в”Ӯ      в””в”Җ prefetch thread fills CircularByteBuffer
                    в”Ӯ
                    в”ңв”Җ BinaryRecordReader (binary deserialisation using FieldDef)
                    в”Ӯ
                    в””в”Җ GenericRowRecordBuilder вҶ’ Spark Row
                                в”Ӯ
                                в–ј

```

Column pruning flows backwards: Spark's pruned scan passes only required field names вҶ’ `ColumnPruner` produces a projected `FieldDef` вҶ’ passed into `HpccRDD` as `projectedRecordDef` вҶ’ `BinaryRecordReader` skips non-projected fields during binary deserialisation.

## 7. Entry Points by Module

### commons-hpcc
No executable entry points. Pure library. Key classes consumers import:
- `org.hpccsystems.commons.ecl.FieldDef` вҖ” schema description
- `org.hpccsystems.commons.ecl.RecordDefinitionTranslator` вҖ” JSON вҶ’ FieldDef
- `org.hpccsystems.commons.errors.HpccFileException` вҖ” primary checked exception
- `org.hpccsystems.commons.ecl.FieldType`, `HpccSrcType` вҖ” type enumerations

### wsclient
Primary entry points:
```java
// Singleton-cached instance
HPCCWsClient client = HPCCWsClient.get(conn);

// Service-specific (lower level, more flexible)
HPCCWsDFUClient dfuClient = HPCCWsDFUClient.get(conn);

// Pooled
HPCCWsClientPool pool = new HPCCWsClientPool(conn);
HPCCWsClient c = pool.checkOut();
```

Connection construction:
```java
Connection conn = new Connection("http", "host", "8010");
conn.setCredentials("user", "pass");
// OR
Connection conn = new Connection("http://user:pass@host:8010");
```

### dfsclient
Primary entry point:
```java
HPCCFile file = new HPCCFile("~myfile::name", conn);
DataPartition[] parts = file.getFilePartsOnCluster("mythor");
HpccRemoteFileReader<MyRecord> reader = new HpccRemoteFileReader<>(
    parts[0], fieldDef, new MyRecordBuilder());
while (reader.hasNext()) { MyRecord r = reader.next(); }
```

### spark-hpcc
Via Spark DataSource API (preferred):
```java
Dataset<Row> df = spark.read()
    .format("hpcc")
    .option("host", "http://127.0.0.1:8010")
    .option("cluster", "mythor")
    .option("path", "/spark/test/dataset")
    .load();
```

Via RDD API (lower level):
```java
HpccFile hFile = new HpccFile("~spark::test::dataset", conn);
HpccRDD rdd = hFile.getRDD(sparkContext);
JavaRDD<Row> jrdd = rdd.toJavaRDD();
```

## 8. Object Pooling Pattern (Detail)

`ObjectPool<T>` (`wsclient/src/main/java/org/hpccsystems/ws/client/utils/ObjectPool.java`) provides the generic mechanism:

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ  ObjectPool<HPCCWsClient>               в”Ӯ
в”Ӯ                                         в”Ӯ
в”Ӯ  unlocked: Hashtable<T, timestamp>      в”Ӯ
в”Ӯ  locked:   Hashtable<T, timestamp>      в”Ӯ
в”Ӯ                                         в”Ӯ
в”Ӯ  checkOut():                            в”Ӯ
в”Ӯ    1. iterate unlocked                  в”Ӯ
в”Ӯ    2. expire if (now - ts) > expiry     в”Ӯ
в”Ӯ    3. validate() вҶ’ if bad, expire       в”Ӯ
в”Ӯ    4. move to locked, return            в”Ӯ
в”Ӯ    5. if none found, create() new       в”Ӯ
в”Ӯ                                         в”Ӯ
в”Ӯ  checkIn(t):                            в”Ӯ
в”Ӯ    1. remove from locked                в”Ӯ
в”Ӯ    2. add to unlocked with now ts       в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

`HPCCWsClientPool` overrides `create()` вҶ’ `new HPCCWsClient(m_connection)`.

A **separate** (non-pool) instance caching mechanism exists via `DataSingletonCollection`:  the static `HPCCWsClient.All` collection acts as a per-connection flyweight store, where `get(conn)` returns the same instance for identical connection parameters. This is a **flyweight** pattern, not a pool вҖ” there is no checkout/checkin.

## 9. WSDL/ESDL Generation vs. Hand-Written Code

### Generated (do not modify)

| Location | How Generated | When Updated |
|---|---|---|
| `wsclient/src/main/java/org/hpccsystems/ws/client/gen/axis2/` | Axis2 WSDL2Java from HPCC ESP WSDL | When HPCC platform ESP interfaces change |
| `wsclient/src/main/java/org/hpccsystems/ws/client/wrappers/gen/` | `Axis2ADBStubWrapperMaker` tool (run from `wsInterfaceUpdater.py`) | After new gen stubs are produced |
| `wsclient/src/main/java/org/hpccsystems/ws/client/antlr/gen/` | ANTLR4 grammar (`EclRecord.g4`) | When ECL record grammar changes |
| `dfsclient/src/main/java/org/hpccsystems/generated/` | Template code generation | Build-time via `templating-maven-plugin` |

### Hand-Written

| Location | Purpose |
|---|---|
| `wsclient/src/main/java/org/hpccsystems/ws/client/*.java` | Service clients (`HPCCWsDFUClient`, etc.) |
| `wsclient/src/main/java/org/hpccsystems/ws/client/utils/` | Connection, ObjectPool, Utils |
| `wsclient/src/main/java/org/hpccsystems/ws/client/platform/` | Platform model (Cluster, Workunit, etc.) |
| `wsclient/src/main/java/org/hpccsystems/ws/client/wrappers/` (root) | Exception wrappers |
| `wsclient/src/main/java/org/hpccsystems/ws/client/wrappers/wsdfu/` | DFU domain wrappers |
| `wsclient/src/main/java/org/hpccsystems/ws/client/wrappers/wsworkunits/` | Workunit domain wrappers |
| `wsclient/src/main/java/org/hpccsystems/ws/client/extended/` | WsAttributes client (hand-crafted) |
| `dfsclient/src/main/java/org/hpccsystems/dfs/` | All DFS client code |
| `commons-hpcc/src/main/java/` | Everything in commons |
| `spark-hpcc/src/main/java/` | Everything in spark connector |

### The `wsInterfaceUpdater.py` Workflow
Located at `wsclient/utils/wsInterfaceUpdater.py`. The overall pipeline:
1. Download WSDL/ESDL definitions from a live or committed HPCC ESP
2. Run Axis2 WSDL2Java to produce raw stub classes вҶ’ `gen/axis2/<service>/<version>/`
3. Run `Axis2ADBStubWrapperMaker` to produce wrapper classes вҶ’ `wrappers/gen/<service>/`
4. Update the `latest` symlink/package alias
5. Update `pom.xml` exclude list for old versions

## 10. OpenTelemetry Integration

`@WithSpan` annotations (from `opentelemetry-instrumentation-annotations`) are used throughout `BaseHPCCWsClient`, `Connection`, and `HPCCWsClientPool`. The `Connection` class:
- Creates a `SpanKind.CLIENT` span for each outbound HTTP call
- Injects W3C trace context headers into SOAP requests via `BaseHPCCWsClient.injectOtelContextIntoHttpRequest`
- `HpccRDD` and `HpccFileWriter` propagate `parentTraceID` / `parentSpanID` strings across Spark executor JVMs

## 11. Test Architecture

Tests are tagged via marker annotation interfaces in `org.hpccsystems.commons.annotations`:

| Annotation | Surefire Group | Purpose |
|---|---|---|
| `@BaseTests` | `groups` (default) | Standard unit tests, always run |
| `@IntegrationTests` | optional | Require live cluster |
| `@KnownServerIssueTests` | excluded by default | Tests with known HPCC server bugs |
| `@UnverifiedServerIssues` | excluded by default | Potential server issues |
| `@UnverifiedClientIssues` | excluded by default | Potential client issues |
| `@RemoteTests` | optional | Require network |

All `wsclient` test classes extend `BaseHPCCWsClientTest` (located in `wsclient/src/test/java/`).  
Spark integration tests extend `BaseIntegrationTest` in `spark-hpcc/src/test/`.
